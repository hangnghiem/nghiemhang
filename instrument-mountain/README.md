# 楽器の山 — Instrument Mountain 🏔️🎻

楽器クイズと開拓シミュレーションを組み合わせたブラウザゲーム。
プレイヤーは開拓団のリーダーとなり、山を登りながら楽器クイズに挑戦。
資源と住民（音楽家）を集め、村を建設して、最終的に山頂の「オーケストラ都市」完成を目指します。

PC・タブレット・スマートフォンに対応したレスポンシブデザインです。

## 遊び方

1. **ゲストプレイ**で名前を入力して冒険開始（データはブラウザの LocalStorage に自動保存）
2. ふもとの村から**登山ルートを選択**（分岐あり）
3. ルート上の**楽器クイズ**に挑戦し、正解で資源・図鑑登録
4. 道中で**資源 / 住民 / 宝箱 / 特殊イベント（音楽祭）**を獲得
5. 山頂に到達して戦利品を村へ持ち帰る
6. 木材・石材・住民が貯まったら**新しい村を建設** → 上のエリアが解放
7. 弦楽器村 → 管楽器村 → 打楽器村 → 民族楽器村 → **オーケストラ都市** を目指す

## 実装済み機能（MVP）

- ✅ ログイン（ゲストプレイ / LocalStorage セーブ・ロード・自動保存）
- ✅ 楽器クイズ **100問**（楽器ジャンルのみ。音楽理論・作曲家・歌手・音楽史は不出題）
  - 🖼️ 画像クイズ / 🌑 シルエットクイズ / 💡 知識クイズ
- ✅ 登山システム（ルート分岐・クイズ・資源・住民発見・宝箱・特殊イベント）
- ✅ 資源システム（基本：木材・石材・食料 / レア：設計図・楽器パーツ・古代楽譜）
- ✅ 住民システム（音楽家ごとのボーナス効果：人気度・クイズ報酬・資源獲得・レアイベント・全体ボーナス）
- ✅ 村建設システム（条件達成で新しい村を建設しエリア解放）
- ✅ 楽器図鑑（40種・カテゴリ別・コンプリート率表示）
- ✅ 実績 / 統計
- ✅ レスポンシブ UI（ヘッダー / 左サイドバー：山マップ / 中央：登山・クイズ / 右サイドバー：図鑑・住民・実績、モバイルはタブ切替）

## ディレクトリ構成

```
src/
  app/                 # Next.js App Router（layout / page / globals.css）
  components/          # UI コンポーネント
    center/            #   中央パネル（村ハブ・登山・クイズ）
  lib/                 # ゲームデータ & ロジック
    instruments.ts     #   楽器マスター（40種）
    questions.ts       #   クイズ100問の生成
    villages.ts        #   村・エリア・住民の定義
    game.ts            #   初期状態・ボーナス計算・登山生成・報酬
    storage.ts         #   LocalStorage セーブ/ロード
    types.ts           #   型定義
  store/
    GameProvider.tsx   # ゲーム状態（Context + Reducer）
```

## 設計メモ

- クイズの選択肢は問題 ID をシードにした決定論的シャッフルで生成し、セーブデータと整合させています。
- 画像・シルエットクイズは絵文字をビジュアルとして使用（シルエットは CSS フィルタで黒塗り）。専用絵文字のある11種から出題し、残りは知識クイズで補完して計100問にしています。
- 収益化方針（基本無料・Pay to Win 禁止）に沿い、ゲーム進行はプレイで得た資源と住民のみに依存します。
